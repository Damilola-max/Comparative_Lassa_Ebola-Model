# ESM-Based Mutation Analysis Pipeline

## Overview
This document outlines the ESM-based mutation analysis pipeline developed for analyzing mutations in Lassa and Ebola viruses. The pipeline is designed to facilitate comprehensive mutation scoring and interpretation.

## Workflow Steps
1. **Data Collection**: Gather relevant genomic sequences.
2. **Preprocessing**: Clean and prepare data for analysis.
3. **Mutation Identification**: Identify mutations present in the sequences.
4. **Mutation Scoring**: Score mutations based on predefined criteria.
5. **Results Visualization**: Visualize key results to interpret findings.

## Notebooks
- **Notebook 1**: Data preprocessing and exploration.
- **Notebook 2**: Mutation identification and scoring.
- **Notebook 3**: Result visualization and interpretation.

## Mutation Scoring System
- Mutations are scored based on factors such as functional impact and prevalence.
- A summary table of mutation scores will be generated for easy reference.

To setup the environment, follow these steps:
1. Clone the repository: `git clone https://github.com/Damilola-max/Comparative_Lassa_Ebola-Model`
2. Install dependencies: `pip install -r requirements.txt`
3. Run the notebooks in your chosen environment.
